User API Documentation

Welcome to the User API! This API allows you to perform CRUD (Create, Read, Update, Delete) operations on user data.

How to Use the API

To use the User API, follow these steps:

  1. Clone the GitHub Repository:
git clone https://github.com/iusm4n/restfulapi

Replace your-username with your GitHub username and your-api-repo with the name of your API repository.

  1. Change Directory:
cd restfulapi

Change to the directory of your cloned repository.

  1. Install Dependencies:
npm install

Install the required dependencies for the API.

  1. Start the Server:
npm start

Run the API server. You should see a message indicating that the server is running.

Now, you can use the provided API endpoints as described in the documentation above.

Get All Users

Request

GET /api/users/

Response

[
  {
    "_id": "user_id_1",
    "name": "John Doe",
    "email": "john.doe@example.com",
    "address": "123 Main Street"
  },
  {
    "_id": "user_id_2",
    "name": "Jane Doe",
    "email": "jane.doe@example.com",
    "address": "456 Oak Avenue"
  },
  // ... other users
]

Get a Specific User

Request

GET /api/users/:email

Response

{
  "_id": "user_id",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "address": "123 Main Street"
}

Update a User

Request

PUT /api/users/:email

Request Body

{
  "name": "New Name",
  "email": "new.email@example.com",
  "address": "New Address"
}

Response

{
  "_id": "user_id",
  "name": "New Name",
  "email": "new.email@example.com",
  "address": "New Address"
}

Delete a User

Request

DELETE /api/users/:email

Response

{
  "_id": "user_id",
  "name": "John Doe",
  "email": "john.doe@example.com",
  "address": "123 Main Street"
}

Create a New User

Request

POST /api/users/

Request Body

{
  "name": "New User",
  "email": "new.user@example.com",
  "address": "New User Address"
}

Response

{
  "_id": "new_user_id",
  "name": "New User",
  "email": "new.user@example.com",
  "address": "New User Address"
}

Feel free to use these examples to interact with the User API and manage user data. If you have any questions or issues, please refer to the API documentation or contact the API administrator.
